flutter 网络请求

# flutter 网络请求

网络请求是app及前端必须必备的，Dart和flutter 也提供了相关库

# http

使用http 提供简单的 方法从网络上获取数据 http 包含一组高级函数和类，它与平台无关，可以在 app和浏览器中使用 可以在 pubspec.yaml 中添加 依赖，添加最新版本的 http

dependencies:
  http: <latest_version>

# 简单的用法

  Future<http.Response> fetchPost() {
    return http.get('xxxxxxxurl');
  }
1
2
3

get 方法 返回一个 包含 Response 的 Future

  • Future是一个用于处理异步操作的核心Dart类。它用于表示将来某个时候可能出现的潜在值或错误
  • http.Response 包含 请求成功 所接受到的数据

如果向同一服务器发出多个请求, 可以单独用client对象，但是到确保完成时关闭client

var client = new http.Client();
try {
  var uriResponse = await client.post('http://example.com/whatsit/create',
      body: {'name': 'doodle', 'color': 'blue'});
  print(await client.get(uriResponse.bodyFields['uri']));
} finally {
  client.close();
}

NOTE: factory 关键字

  • 工厂构造函数不能访问 this.
  • 工厂模式创建 一个工厂构造函数可能从缓存中返回实例或返回一个子类的实例.

# json serialization

  1. Manual serialization (手工序列化) 手动JSON解码是指在dart:convert中使用内置的JSON解码器。它包括将原始JSON字符串传递给jsonDecode()函数，然后在生成的Map<string, dynamic>中查找所需的值。它没有外部依赖关系或特定的设置过程，并且对于概念的快速证明非常有用。 手动解码的缺点 非常明显，当项目变大时。手动解码效率很低，如果 json字段拼错，将导致代码在运行时错误。

  2. Automated serialization using code generation (使用代码生成的自动序列化)

带有代码生成的JSON序列化意味着有一个外部三方库为您生成编码样板。在进行一些初始设置之后，运行一个文件监视程序，该程序从模型类生成代码。例如，json_serializable和built_value就是这类库。

json_serializable 的使用 首先安装 依赖 在 pubspec.yaml

dependencies:
  # Your other regular dependencies here
  json_annotation: ^2.0.0

dev_dependencies:
  # Your other dev_dependencies here
  build_runner: ^1.0.0
  json_serializable: ^2.0.0

使用：

import 'package:json_annotation/json_annotation.dart';

/// This allows the `User` class to access private members in
/// the generated file. The value for this is *.g.dart, where
/// the star denotes the source file name.
part 'user.g.dart';

/// An annotation for the code generator to know that this class needs the
/// JSON serialization logic to be generated.
@JsonSerializable()

class User {
  User(this.name, this.email);

  String name;
  String email;

  /// A necessary factory constructor for creating a new User instance
  /// from a map. Pass the map to the generated `_$UserFromJson()` constructor.
  /// The constructor is named after the source class, in this case User.
  factory User.fromJson(Map<String, dynamic> json) => _$UserFromJson(json);

  /// `toJson` is the convention for a class to declare support for serialization
  /// to JSON. The implementation simply calls the private, generated
  /// helper method `_$UserToJson`.
  Map<String, dynamic> toJson() => _$UserToJson(this);

  /// Tell json_serializable that "registration_date_millis" should be
/// mapped to this property.
/// 可以 通过 @JsonKey定制 model 的名字
@JsonKey(name: 'registration_date_millis')
final int registrationDateMillis;
}

# 深入分析

Future 抽象类实现 需要注意几点：

  1. Dart 是基于单线程模型的语言
  2. Future 对象 是 一系列操作的结果
  3. 若要在 future 完成之前暂停执行，请在异步函数中使用wait。
  4. 要捕获错误，请在异步函数中使用try-catch表达式。

异步操作允许程序在等待操作完成时完成其他工作。Dart使用Future对象(Future)表示异步操作的结果。要处理Future，您可以使用async and waiting或Future API。

future是一个future 对象，它表示产生T类型结果的异步操作。如果结果不是一个可用值，那么future的类型是future。当调用返回future的函数时，会发生两件事:

  1. 该函数对要完成的工作进行排队，并返回一个未完成的将来对象。
  2. 稍后，当操作完成时，Future对象将使用一个值或一个错误完成。 future 一般是在 Use async and await 或者 Future API 中使用 有点想js 中promise.
